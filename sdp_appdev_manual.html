Overview

ServiceDesk Plus (SDP) is an ITSM platform used for managing service requests, incidents, changes, and assets. This manual is tailored for the App Dev group, focusing on workflows, automation, and integrations relevant to application development and support.

Login & Access

  1. Navigate to https://servicedeskplus.company.com.
  2. Login using your corporate credentials (SSO or username/password).
  3. For access issues, contact IT support or use the password reset feature.
Note: App Dev group members have elevated permissions for ticket categories related to application development.

Ticket Management

Creating a Ticket

  1. Click New Request from the dashboard.
  2. Select the appropriate category (e.g., Application Bug, Feature Request).
  3. Fill in required fields: Subject, Description, Priority, Application Name.
  4. Attach relevant files or screenshots.
  5. Click Submit.

Updating & Resolving Tickets

  • Open the ticket from My Requests or Assigned to Me.
  • Add notes, change status, or update fields as needed.
  • To resolve, set status to Resolved and add a resolution note.

Ticket Lifecycle

New → Assigned → In Progress → On Hold → Resolved → Closed

Automation & Integrations

Custom Workflows

  • Automated assignment based on application and priority.
  • Email notifications for status changes.
  • Auto-close tickets after resolution confirmation.

API Usage

# Example: Create a ticket via SDP API (Python)
import requests

url = "https://servicedeskplus.company.com/api/v3/requests"
headers = {"Authorization": "Bearer <your_api_token>"}
data = {
    "subject": "App Bug: Login Failure",
    "description": "Users unable to login to AppX.",
    "category": "Application Bug",
    "priority": "High"
}
response = requests.post(url, headers=headers, json=data)
print(response.json())
Tip: For API tokens and integration setup, contact the SDP admin.

FAQ

How do I escalate a ticket?

Change the priority to Critical and add an escalation note. Notify the team lead if urgent.

Where can I find API documentation?

Visit SDP API Docs or ask the admin for access.

Who do I contact for workflow changes?

Submit a request under Workflow Change or email the SDP admin team.
